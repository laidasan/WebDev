My Note

Fixed

What?

先前有介紹過了relative的相對定位方式，其實不只有relative這種定位模式而已(有三種)，今天來介紹"fixed"的定位方式。

How?

fixed在三種模式裡面比較特別的，在這個頁面上右上角的漢堡(導覽列)就是用fixed定位住的，你會發現不管螢幕怎麼滑動，它都是固定在那個位置，這裡我們就知道，fixed會依照視窗固定在視窗的某個位置上，我們再來看看其他的例子: ←Codepen

這裡有一堆段落，中間有一個灰色的區塊，之前有說過div是老大大哥，所以它會佔據整行(即使寬度不到整行)，而當我們對它做fixed定位的時候它會它生什麼事情呢?! ←Codepen

你會發現!!唉唷，Fixed好像飄起來一樣，而且蓋住了底下的段落文字了。這就是上了fixed的效果，以圖層來說就好像把"資料抽出來"變成獨立的一層圖層，接下來我們來設定一些top、left、right、bottom， 假設我想要讓這個方塊在我螢幕上的左上角的話，我可以這麼做: ←Codepen

我把margin註解掉後，並設置了"top: 12px;"與"left: 30px;"，這個意思就是說"距離我視窗頂部12px"、"距離我視窗左邊30px"的意思，這時你在麼麼去滑動視窗，它都還是固定在左上角，那這時問大家一個問題，如果我只設定了fixed，而沒設定top、left、bottom、right的話，位置要怎麼去定位? 可以去看看第二個Codepen連結(還沒有設置top等等的時候)。

答案是: 會定位在"原本資料"在視窗的位置上

其實fixed很常非常常看到，就是當你在看些文章的時候，有時很令人討厭的事情——蓋板廣告!!!!!!!

是不是回想起來了?!!當我們用手機或用電腦看文章的時候，一點進去就會先跳出蓋住整個版面的廣告，關掉後才能夠觀看或是較方便滑動觀看，那個蓋板廣告就是用應用fixed的定位模式喔!!

  • ※當position設定為fixed模式的時候，會做的第一件事情就是，把資料抽離、抽出(像是第二個codepen那樣，像是抽出來做一個額外獨立的圖層)
  • ※抽離後的位置如果沒又去做設定(top left right bottom)就會在資料原本的位置上
  • ※fixed的定位一定會在可視範圍內視窗之內

End

fixed就先介紹到這裡，就是這麼的簡單!!但也很常運用到喔!到現在為止我們談了fixed與relative這兩種模式了，接下來就讓我們在另外一篇談談"absolute"絕對定位模式吧!!
