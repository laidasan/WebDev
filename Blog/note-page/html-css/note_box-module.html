Box Module

What?

今天來談談區塊尺寸計算的方式，在區塊尺寸的計算方面我們會有專有名詞——"Box Module"，也就是盒模型。 我們時常在設定width、height，那我們設定的寬度與高度到底是設定給誰的阿?而我們設定寬度與高度有什麼樣的重要性?

對於排版來說，假如今天我們要做兩欄式、三欄式的排版時候，我們就需要去計算寬度讓排版正確的顯示，很有可能差一點點就會導致整個 排版壞掉!

How?

假設今天我們設定一個區塊，寬度與高度給它設為300px。

第一個div

最一開始學習的時候，我很直觀的去認為:「喔!整個區塊的寬度與高度就是300px!」但事實上真的是這個樣子嗎? 我們再加上padding(內距)為20px來比較，看看會不會有差別!

第二個div

咦?!為什麼明明寬度設定的是300px,有padding與沒padding的區塊為什麼寬度會不一樣啊?! 我們再設border(邊框)為10px看看!!

第二個div
第三個div

咦?!怎麼又不一樣了?其實在一個區塊的時候寬度與高度確實是300px,而當我們設定內距(第二個div)或是邊框(第三個div) 時，整個寬度如下: 第一個:300px = 300px 第二個:20px+ 300px + 20px = 340px (padding:20px;) 第三個:10px +20px + 300px + 20px +10px = 360px (padding:20px border:10px)

其實一個區塊的"尺寸"是由三個去組成的，分別是

  • 內文
  • 內距
  • 邊框

以上是我們看的到尺寸，咦?看得到?那是不是還有看不到的?是的，我們還有一個margin(外距)，也就是跟向外跟其他元素的距離， "外距 + 內距 +內文 + 邊框"才是一個區塊(元素)佔據的"空間"。 ※以上預設狀況的時候

現在我們現在要做個三欄式的畫面，每一欄都"佔據"200px的寬度的話，我們就必須要去計算了。 假設我內距要設成12px,邊框1px,每一欄都間隔10px,那我的width就要設為: 200 - 12*2 - 1*2 -20 = 154px;
