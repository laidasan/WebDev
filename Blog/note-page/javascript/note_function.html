function

What?

在上一篇物件與陣列裡或是在更之前的文章裡面會看到function...等等的函式，今天就來介紹「function」到底是什麼東西來著!!

function宣告

我們可以這樣有名字的宣告:

function test(){

console.log('test');

}

通常我們為需要用到函式的時候，就是需要重複執行某些內容的時候，就猶如撰寫Java把重複的內容封裝成一個方法，優化程式碼，function也可以做到這樣的事情，除此之外，函式也可以傳入引數，比如說:

function test(num1,num2){

return num1 + num2;

}

console.log(test(10,20) );

這邊可以注意一下，如果function沒有return東西的話，那麼在就會回傳undefined，但上們我們有return num1 + num2，所以console出來是30唷!可以試試看console出沒有return的function(return undefined)!

我們再來試一下，如果傳入的引數只有一個呢?

function test(num1,num2){

return num1 + num2;

}

console.log(test(10) );

我們會得到「NaN」，奇怪勒?會什麼會是NaN呀!我們把num1與num2都console出來看看!

function test(num1,num2){

console.log(num1);

console.log(num2);

}

console.log(test(10) );

console出來的結果會是「10」還有「undefined」，也就是說傳入較少的引數是可行的，只不過不足的部分是undefined，那如果傳入較多的引數呢?比如說:

function test(num1,num2){

console.log(num1);

console.log(num2);

}

console.log(test(10,20,30) );

會發現程式還是跑得起來，不過出來的結果是「10」與「20」，奇怪了!那第三個引數跑哪去了呢?

其實，在函式內會自動宣告一個arguments名稱參考至具陣列外觀的物件(之前有提過javascript裡沒有真正的陣列(Array)物件，只不過是外觀相像而已)，而它帶有所有傳入的引數，所以我們可以這樣:

function test(num1,num2){

for(let i = 0;i < arguments.length;i++){

console.log(arguments[i]);

}

}

test(10,20,30);

這樣就會看到我們所有傳入的引數囉!!
