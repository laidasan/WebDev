運算

What?

上一篇我們有聊到邏輯運算與比較運算的部分，最後說了==與===的差別，不過都只在基本型態上而已，今天會來談談如果是在物件的比較或是邏輯運算上會如何處理!!

How?

除了+,-,*,/與==,===外，其實還有>=,>,<,<=,!=,!==這些運算，驚嘆號是相反的意思，!=也就是不等於的意思。

上一篇我們都是用基本型態來做運算，現在我們來試試看用物件形態上的運算吧!在物件的運算上，我們會需要兩種特性，一個是valueOf,一個是toString，valueOf會是在用在轉換成數字時用到，而toString就是轉換成字串的時候拉!比如說:

const person = {

name: 'joe',

age: 40,

valueOf: function() {

return 40;

}

}

let num = 20;

console.log(person + num);

console.log(person > num);

在上面的例子裡面，在person裡面設定了一個valeOf特性，為一個function且回傳40,最後的兩個console出來的結果為"60"與"true"，在第一個做加法的時候，會嘗試轉換成數值，嘗試使用valueOf方法，所以回傳為40，而40+20=60，第二個依此類推。

至於toString

const person = {

name: 'joe',

age: 40,

toString: function() {

return 'My name is ' + this.name;

}

}

console.log('Hi,' + person);

出來的結果是"Hi,My name is joe"。

要注意的一件事:valueOf()預設會回傳toString()的回傳值，所以如果沒有重新定義valueOf的話就會使用toString的回傳值。

這邊我們在提一個事情，如果在基本型態時做>=,>,<,<=的時候，一邊是字串一邊是數字的時候要怎麼辦，這個時候字串如果部分為數字的話，則會轉換成數字在進行比較，如果字串都沒有數字的話則會轉換成NaN，如果兩邊都是字串，則會逐字元依Unicode編碼值進行比較。

※這邊還很不了解的是，當valueOf與toString同時存在的時候，都會使用到valueOf的方法，還需要去了解背後原因。

接下來我們來講&&與||吧!，直接用例子來說明更清楚!

let num1 = 10;

let num2 - 20;

let num3 = 30;

if( num3 > num1 && num3 > num2){

console.log('num3 is biggest');

}else {

console.log('oh no!');

}

出來的結果是"num3 is biggest"喔!在這裡&&要兩邊都為true才回傳true喔!先看左邊num3 > num1，答案是true所以繼續往右邊看num3 > num2，結果也為true，所以&&的結果就回傳true囉!!

而||只要其中一邊是true就會回傳true囉!以最簡單的白話來講，假如小明去圖書館的條件是"禮拜三，而且要是個大晴天"才去，而小華則是"禮拜三，或是大晴天"，小明就很像是&&，要禮拜三"且"又是大晴天才去，而小華就像||，禮拜三"或者"大晴天。

在直觀的直覺上是這樣子沒錯，但對電腦運算來說是有真面目的，在進行&&的時候，會先從最左邊開始判斷，如果是true就繼續往下一個(右邊)進行判斷，直到判斷全結束或是判斷到某個false就停住，比如說:

console.log(20 && 30 && 10);

這裡console出來的結果會是10，為何呢?之前我們有提到過在javascript中任何東西都可以轉換成boolean值，除了false family之外的都是true，而依照剛剛&&邏輯來判斷，首先先從最左邊30來看，是true，再來看20，是true,再來看10，是true，所有判斷都結束了，指標指在10上，所以回傳10!再來看下面這個:

console.log(30 && undefined && 10);

這個conosle出來的結果是undefined喔!!我們再來看一次，首先先從最左邊30開始看，true，看下一個，undefined為false，判斷就到此為止，不會繼續往下判斷了(可以把他想像為因為&&都要是true才是對的，那麼是不是找到一個false就代表過不了拉!!所以如果找到false就直接停止就好了)，而停下後判斷指標在undefined上，所以就回傳undefined，所以console的結果是undefined!

至於||剛好是與&&相反的唷!只要找到一個true就會停止往下判斷並回傳，比如說:

console.log(undefined || 20 || null);

這裡console出來的結果是20，首先先從最左邊開始判斷，undefined為false所以繼續往下，到了20為true，所以停止往下繼續判斷並回傳。

我們可以運用這個技巧來給預設值，比如說:

function show(x) {

return x || 20;

}

console.log(show(100));

console.log(show());

第一個結果是100，而第二個結果是20，在show這個function中如果我們有給X的話就會回傳X，如果沒有給的話X就為undefined，繼續往下判斷20為true，所以回傳20。我們會很常使用這個方法當使用者沒有給值的時候，我們就會給預設值的時候，是個很方便的技巧!!
