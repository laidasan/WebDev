My Note

FLOAT

WHAT?

float是一個較老舊、傳統的排版方式，現在也有了flex、grid比較新的技術方式，那我們可能會想:已經有了flex或是grid 比較新也很好用的方式，我們還需要學習怎麼使用比較老舊傳統的float嗎?

雖然已經有了比較新的技術方式，但我們在維護網站的時候也很有可能會遇到以前開發的網站(較舊)，或許那時候還沒有flex與grid， 所以是用float去做排版的，這個時候我們就會需要使用到拉!! 事不宜遲，讓我們來進入這篇重點吧!

HOW?

我們先來講講它的原理，先想像在一個平面上有我們的容器區塊，而容器裡面有我們的圖片、段落等等。 這裡我們用下面這兩條線來解釋會更清楚，從側面來看。

紅色的為img,黃色的為段落(文字)

上面原本是兩個卡得好好的，沒有什麼問題。當我們把img的float值設為left的時候，也就是使img浮起來。

這時候段落(黃色)部分會因為img(紅色)浮起來，前面沒有東西可以卡住了，所以就會往前，卡到能卡住的地方。

clear-fix

但是float會有個問題，就是當我們用float的時候，原本是父層容器的高度是靠子層來撐開的話，那麼父層的容器高度就會崩塌。 →CodePen 點擊旁邊的codepen後我們來看看，先把Css中.col的float屬性註解掉，就會出現父層容器原本的樣貌，但是將float屬性加上去之後， 就會發現:咦?!父層怎麼不見了!!!這個就是父層高度崩塌的現象，那我們要如何解決呢?!

那就是搭配"clear"屬性拉!!

※要看看原本不見的父層在哪裡的話，可以把container裡面的padding註解拿掉就看的比較清楚囉!!

我們float之後，在最後一個float的區塊後面添增一個區塊，這個區塊使用clear屬性來清除浮動，在"CodePen"裡我們將html的最後一個區塊註解拿掉， 這樣就可以將父層的高度正常顯示拉!!看看css裡面的clearfix，這個class將clear屬性的值設為both(left|right|both|none)。 或者是可以利用偽元素來寫。 .container::after { content: ' '; clear: both; display: table; }

雖然說是清除浮動，但也可說閃開浮動物件，跑到浮動物件的後方(不會浮動就卡不住)就像下圖這樣就不會往前了。 (沒有設clear的話就會往前)
